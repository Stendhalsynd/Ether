# SPRINT 1 DELIVERABLES (Iteration 1)

## 1) Run Engine Core (`T1`)
- 파일/구간: `RunEngine` 최소 규격
- 인터페이스:
  - `createRun(seed, party, objectives, loadouts): RunContext`
  - `tickRun(runId, action): RunEvent[]`
  - `exitRun(runId): ChronicleSnapshot`
  - `resumeRun(runId, restoreSnapshot): RunContext`
- 출력물: 런 컨텍스트 스키마(최소 12개 필드), 종료 시 자동 요약 키워드

## 2) Risk Stack 정합성 (`T2`)
- 오염/항체/정화 상호작용 규칙 v1:
  - `에테르 노출 + 오염 진행`
  - `혈석 사용 = 감염 지연, 대가 증가`
  - `성석 발동 = 지역 정화, 생태 반응 악화 가능성`
- 출력물: 정량 규칙표(피크/누적/회복률), 상호작용 우선순위 규칙

## 3) 프롤로그/초기 플롯 전환 (`T4`)
- 전이 트리거:
  - 기초 생존 완주 + 첫 에테르 징후 인지 -> `Act0 완료`
  - 최초 로미아스 제약 반응 획득 -> `Act1 잠금 해제`
- 출력물: 전이 조건표 + 실패 복구 조건(죽음/복귀/리트라이)

## 4) NPC 상태 기계 초안 (`T5`)
- 상태 타입: `공감`, `의심`, `거래`, `적대`, `중재`
- 전환 규칙:
  - 로미아스: 생존훈련 완료 시 `의심` 증가 억제
  - 라네이레: 증거량 구간별로 `신뢰도` 산출
  - 사이모어: 영웅화 누적 시 `이용` 상태 강화
  - 세비리스: 정보 유통량/대가 기반 `반전` 확률
- 출력물: 상태 전이표 + 핵심 대사 훅 3개

## 5) Chronicle/Proof Board UI 요구사항 (`T12`, `T13`)
- Chronicle:
  - 런 종료 시 문장형 요약 5~8줄
  - 항목: 생존지표/선택/손실/증거/보상
- Proof Board:
  - 카테고리: `오염`, `항체`, `거래`, `증거`, `결정`
  - 증거 클릭 시 `관련 지역`, `관련 NPC`, `메인 플롯 연결점` 노출
- 출력물: 화면 컴포넌트 항목 정의 + 상태 전이 규약

## 6) QA 시나리오 (S1, S2)
- S1: 초심자 런 20분 내 기초 생존 경로 단절 없음 검증
- S2: 같은 증거 조합으로 라네이레 상태가 단계적 업데이트되는지 확인
- 출력물: 수동/자동 체크리스트 템플릿(통과/실패/블로커 항목)
