# SPRINT 2~4 DELIVERABLES (Iteration 2~4)

## SPRINT 2 (Iteration 2)
### T3: Stage Templates
- 5개 템플릿 최소 스펙:
  - 생존형, 정화형, 거래형, 탐사형, 의사결정형
- 공통 메타: 난이도, 리스크 게이지, 보상 임계치

### T6: 라네이레 진실 해석 갱신
- 증거 임계치별 해설 단계 3단계:
  - 1차 단서: 경고성 문장
  - 2차 단서: 대립 서사 상충
  - 3차 단서: 의도 불명확성 노출

### T7: 엔딩 분기 v1
- 분기 라벨: 보존형, 제어형, 봉인재작성형, 균열수용형
- 최소 공통 조건: 진실 수집량, 동맹 신뢰도, 시즌 상태, 선택 기록

### T8 / T9: 사건 모듈 + 증거 반영
- 사건 카탈로그:
  - 오염 증거, 항체 대가, 진실 거래, 선택 기록
- 반영 규칙:
  - 사건 결과를 `Proof Board`에 증거 토큰으로 저장
  - 일정량 누적 시 라네이레/시즌 브릿지 업데이트

### FE: T14 / T15
- 시즌 예고 UI:
  - 예보 신뢰도(낮음/중간/높음) 표시
  - 긴급성 아이콘 + 리스크 요약
- HUD:
  - 오염, 정화, 진실 신호를 1개 패널로 통합

### QA: S3 / S4
- S3: 연속 3런 이벤트 가중치 상호작용 검증
- S4: 시즌 전환 직후 네피아 텍스처/이벤트셋 변경 적용 검증

## SPRINT 3 (Iteration 3)
### T10: 선택 기록 기반 가중치
- 플레이 기록이 다음 런 이벤트 생성 시 반영되는 규칙 확정
- 반감/증폭 계수 상한선 정의

### T11: 시즌·펙트 보정
- 시즌 상태에서 상호작용 보정이 룬/지역/동료별로 다르게 적용되도록 규칙화

### T16: 시즌 상태 머신
- 상태: 개막/확산/결산/안정화
- 전환 조건: 이벤트 카운트, 플레이어 참여율, 시즌 보스 진행률

### T17 / T18: 변형 세트/시즌 보스
- 변형 세트: 환경효과, 상태이상, 몬스터 군집
- 시즌 보스:
  - 클리어만이 아닌 협업/의식형 성공조건 병행

### QA: S5
- 4개 엔드 분기 연결 점검
- 시즌 1종 종료 시점의 회귀 시나리오 산출

## SPRINT 4 (Iteration 4)
### 안정화 항목
- FE: 모바일 가독성, 터치 우선순위, 내비게이션 회귀 테스트
- BE: 저장 복구 경로, 데이터 정합성, 실패 복구 시퀀스
- Planner: 플롯 충돌 감지 리스트(진실 모순, NPC 동기 충돌, 지역 스크립트 충돌)
- QE: 통합 회귀 세트 집계(시나리오 기반)
- Validator: 승인 기준 최종 점검 표 작성

### Sprint4 종료 판정 기준
- blocked 항목 = 0
- P0 미완료 = 0
- 스프린트 1~3 산출물 승인 참조 정합성 통과
- 다음 단계(라이브 확장)로 이어지는 이행 문서 존재
